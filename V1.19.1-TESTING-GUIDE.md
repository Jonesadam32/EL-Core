## KNOWN LIMITATIONS (v1.19.1)

These are intentional — they are not bugs:

- **No color wheel picker** — hex codes are entered as plain text. Visual pickers are coming in a future release.
- **No AI logo analysis on admin page** — this will be part of the client portal workflow in Phase 2G-B.
- **Dark mode checkbox records intent only** — no dark CSS is generated yet.
- **`ai_palette_suggestions` and `palette_selected` fields** — these exist in the database but are not editable here. They will be populated by the client portal workflow.

## IF SOMETHING IS BROKEN

| Symptom | Likely cause | Fix |
|---|---|---|
| Brand page shows PHP error | Settings field mismatch | Check `class-settings.php` has all brand fields in `$defaults` |
| Color swatches don't update live | JS error in `admin.js` | Check console; ensure `el-swatch-primary/secondary/accent` IDs match |
| `--el-primary` missing from `<head>` | `inject_brand_variables()` not firing | Check `class-asset-loader.php` is hooking to `wp_head` correctly |
| Upload button doesn't open media library | `wp.media` not loaded | Confirm `wp_enqueue_media()` is called on the brand page hook |
| Saved settings don't persist | Form `name` attributes wrong | All brand fields must be `el_core_brand[field_name]` |
| Expand Site portal has broken colors | CSS variable name mismatch | Compare `--el-*` variable names in asset-loader vs portal template |
