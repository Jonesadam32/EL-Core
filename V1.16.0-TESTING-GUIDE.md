# v1.16.0 Testing Guide — Proposal Narrative Redesign

> **Time needed:** 10–15 minutes
> **Staging site:** https://qd19d0iehj-staging.wpdns.site/wp-admin
> **What this version changes:** Proposals are now flowing narrative prose instead of labeled form fields. The AI writes 5 custom sections, and the client portal shows them as a polished document.

---

## STEP 1: Upload the Plugin

1. Open your staging site WP Admin
2. Go to **Plugins → Add New → Upload Plugin**
3. Click **Choose File** and select `el-core.zip` from `C:\Github\EL Core\`
4. Click **Install Now**
5. When prompted, click **Replace active with uploaded**

**Check:** You see "Plugin updated successfully." If you see errors, stop and tell me exactly what they say.

6. Go to **Plugins** list and confirm EL Core shows **Version 1.16.0**

---

## STEP 2: Verify Existing Features Still Work

Before testing the new stuff, make sure nothing broke.

1. Go to **EL Core → Expand Site** — project list should load normally
2. Click into any existing project — tabs should all appear (Overview, Stakeholders, Discovery, Proposals, Stage History, etc.)
3. Click the **Discovery** tab — if this project had a locked definition, it should still show the green "Definition Locked" banner and read-only fields

**Check:** Everything from v1.14.7 still works. If anything is broken, stop here.

---

## STEP 3: Set Up a Test Project (if needed)

You need a project that has:
- A **locked project definition** (the green "Definition Locked" banner on the Discovery tab)
- A **transcript** that was processed with AI

If you already have one from previous testing, use that and skip to Step 4.

If not:
1. Create a new project (any name, any client)
2. Add yourself as a stakeholder with the **Decision Maker** role
3. Go to the Discovery tab, paste the sample transcript below, and click **Process with AI**
4. Review the extracted data, then click **Confirm & Lock Definition**

**Sample transcript to paste:**
```
Discovery Call - Riverside Youth Academy
Date: February 20, 2026

Client: Maria Gonzalez, Executive Director
Present: David Kim (Board Chair), Angela Wright (Program Manager)

Maria: We need a website that tells our story. We serve about 200 kids in after-school
programs across three sites. Our current site is just a single page on Wix.

David: The board wants something that helps with fundraising. We need a donate button,
event pages for our annual gala, and a way for parents to see what programs are available.

Angela: I need a parent portal where families can see their child's schedule and
communicate with program staff. Right now everything is through group texts.

Maria: Our budget is around $8,000 to $12,000. We're a 501(c)(3) so we want something
that looks professional enough for grant applications.
```

## STEP 4: Create a New Proposal

1. Go into your test project
2. Click the **Proposals** tab
3. Click **"New Proposal"**
4. You should see a new proposal appear in the list with status **Draft**

**Check:** New proposal row appears. If you get an error, note the exact message.

---

## STEP 5: Generate the Narrative with AI

This is the big new feature in v1.16.0.

1. Click **Edit** on the draft proposal
2. The edit modal should open with **5 textarea fields**:
   - **The Situation** — what the client is dealing with now
   - **What We're Building** — what ELS will create
   - **Why Expanded Learning Solutions** — why you're the right partner
   - **Your Investment** — pricing context and what's included
   - **What Happens Next** — the process after signing
3. Click **"Generate with AI"**
4. Wait 10–15 seconds — the button should show "Generating..."

**Check:** All 5 fields populate with flowing narrative prose. It should read like a custom letter, not bullet points or labeled fields.

**If AI generation fails:**
- You'll see an error message — note exactly what it says
- Check: Is the project definition locked? (If not, you should see: "Lock the project definition before generating a proposal.")
- Check: Is the OpenAI API key configured? (EL Core → Settings → AI Integration)
- Check browser console (F12 → Console tab) for detailed errors

---

## STEP 6: Edit the Proposal

1. With the modal still open, read through the AI-generated text in each section
2. In the **"Your Investment"** section, change the price to something specific — for example, change whatever the AI wrote to "$9,500"
3. In the **"The Situation"** section, add a sentence at the end — something like "The board has also expressed interest in a Spanish-language version of the site."
4. Click **"Save Proposal"**

**Check:** You should see a success message. The modal closes or updates.

5. Click **Edit** on the same proposal again

**Check:** Your changes are still there — the $9,500 and the added sentence should both persist. If the fields reverted to blank or to the AI-generated text, that's a bug.

---

## STEP 7: Set the Price

1. While editing, look for the **pricing fields** (budget low, budget high, and/or final price)
2. Set the **Final Price** to `9500` (or whatever number you chose)
3. Click **Save Proposal**

**Check:** Price saves correctly.

---

## STEP 8: Send the Proposal

1. Back on the Proposals tab, find your draft proposal
2. Click **"Send"** (or "Send Proposal")
3. Confirm when prompted

**Check:** Proposal status changes from **Draft** to **Sent**.

---

## STEP 9: View Proposal as Client in the Portal

This is the second big thing to test — how the proposal looks to the client.

1. You need to view the client portal as the Decision Maker. Two ways to do this:
   - **Option A:** If you set up "Login As" (from v1.10.6), go to the project's Stakeholders tab, find the Decision Maker, and click "Login As"
   - **Option B:** Open a private/incognito browser window and log in as the Decision Maker user directly
2. Navigate to the page with the `[el_expand_site_portal]` shortcode (the client portal page)

**What you should see:**

The proposal should render as a **professional document** with:
- **Letterhead header** — "Expanded Learning Solutions" at the top with proposal number, date, and "Prepared for: [client name]"
- **Proposal title** as a large heading
- **5 narrative sections** with indigo uppercase headers:
  - THE SITUATION
  - WHAT WE'RE BUILDING
  - WHY EXPANDED LEARNING SOLUTIONS
  - YOUR INVESTMENT
  - WHAT HAPPENS NEXT
- **Pricing box** — shows the dollar amount you entered, cleanly formatted
- **Social proof** — "Trusted by organizations including:" with placeholder org names
- **Terms & Conditions** — collapsible section at the bottom (click to expand)
- **Accept / Decline buttons** at the bottom (only visible to Decision Maker)

**Check:** The document looks like a professional letter, not a form with labels. The text flows as paragraphs. The Georgia serif font gives it a polished feel.

**If nothing shows:** Make sure the proposal status is "sent" and the logged-in user is a stakeholder (Decision Maker or Contributor) on the project.

---

## STEP 10: Accept the Proposal

1. Still logged in as the Decision Maker in the portal
2. Click **"Accept Proposal"**
3. Confirm when prompted

**Check these three things:**
1. Proposal status changes to **Accepted** — the Accept/Decline buttons disappear and you see "Accepted on [date]"
2. The **project stage advances** — if the project was on Stage 3 (Scope Lock), it should now show Stage 4
3. Go back to WP Admin → Expand Site → open the project — the **Final Price** from the accepted proposal should now be saved on the project record

---

## STEP 11: Test the Error Case

**Check:** You should see an error message like: "Lock the project definition before generating a proposal." The AI should NOT generate anything without a locked definition.

## COMMON ISSUES

**"Generate with AI" button does nothing:**
- Check browser console (F12 → Console) for JavaScript errors
- Make sure the project definition is locked (green banner on Discovery tab)

**Proposal doesn't show in client portal:**
- Proposal must be in "sent" or "accepted" status
- User must be a stakeholder on the project
- Make sure you're on the right portal page (the one with `[el_expand_site_portal]` shortcode)

**Price shows as $0.00:**
- The Final Price field must be filled in before sending
- If only budget_low and budget_high are set, it shows a range instead

**Terms & Conditions section is missing:**
- The T&C field on the proposal is currently blank by default (v1.17.0 will fix this by auto-populating defaults)
- You can manually type T&C text into the proposal edit form to test the collapsible display

**Stage doesn't advance after accepting:**
- Project must be on Stage 3 (Scope Lock) for automatic advancement
- If on a different stage, acceptance still records but doesn't auto-advance
