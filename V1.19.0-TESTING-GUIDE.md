# EL Core v1.19.0 — Testing Guide
## Phase 2G: Branding Workflow

**Upload:** `el-core.zip` → WordPress Admin → Plugins → Add New → Upload Plugin  
**After upload:** Deactivate → Reactivate the plugin (forces settings migration)  
**Test page:** EL Core → Brand (in the WordPress admin sidebar)

---

Before starting, confirm:

## KNOWN LIMITATIONS (by design in v1.19.0)

- **No dark CSS generated** — the Dark Mode checkbox records intent only. Dark styles are a future version.
- **No Google Fonts loading** — custom font names are stored but not auto-loaded. The theme must load the font.
- **No PDF brand kit export** — this is a future feature.
- **Vision AI is Anthropic-only** — OpenAI logo analysis is not supported in this version.

## IF SOMETHING GOES WRONG

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Brand page is blank / white screen | PHP error in settings-brand.php | Check WP debug log; look for syntax errors |
| Colors not saving | Field name mismatch (missing `el_core_brand[...]` prefix) | Check browser Network tab → form payload |
| Pickr doesn't appear | Pickr CDN blocked or not loaded | Check browser console for script errors |
| "Analyze Logo" shows error | AI API key not set, or image URL is a redirect | Verify API key in AI Configuration; try a direct .jpg URL |
| Semantic preview doesn't update | Primary color input ID mismatch | Check JS console for errors |
| Frontend still shows 5 tokens | Old cached CSS | Hard refresh (Ctrl+Shift+R) or clear WP cache |
| Palette cards don't re-appear after reload | `ai_palette_suggestions` not saving | Check Network tab → the hidden input must contain JSON |
