# EL Core v1.27.0 — Testing Guide

> **Upload first:** Use `el-core-v1.27.0.zip` from your Downloads folder.
> WordPress Admin → Plugins → Add New → Upload Plugin → replace existing.
> Confirm "Version: 1.27.0" appears in the Plugins list before starting.

---

## PART 1 — Setup (Do this before any test)

- [ ] Create a WordPress page called **"Client Dashboard"** with content: `[el_client_dashboard]`
- [ ] Make sure a page exists with `[el_expand_site_portal]` (the existing portal page)
- [ ] Make sure a page exists with `[el_client_invoices]` (the existing invoices page)
- [ ] Create (or confirm you have) a test project in Expand Site with a definition already processed from a transcript
- [ ] Create a test client WP user (role: Subscriber) — this is your "Decision Maker" test account
- [ ] In the project, set that user as the **Decision Maker** (either via the project edit form or Stakeholders tab)
- [ ] In EL Core → Clients, make sure that user is listed as a contact on an organization. If not, add them as a contact on the client's organization.

---

## PART 2 — Client Dashboard (`[el_client_dashboard]`)

### 2A — Logged-out state
- [ ] Visit the Dashboard page while logged out
- [ ] Confirm you see a "Please log in" message with a Log In link

### 2B — No org linked
- [ ] Log in as a WP user who has **no entry** in `el_contacts`
- [ ] Visit the Dashboard page
- [ ] Confirm you see: "No client account is linked to your user. Please contact your project manager."

### 2C — Decision Maker sees their project
- [ ] Log in as the Decision Maker test account
- [ ] Visit the Dashboard page
- [ ] Confirm a project card appears for their project
- [ ] Confirm the card shows:
  - [ ] "Website Development" module badge (indigo/purple)
  - [ ] Stage number and stage name (e.g. "Stage 1: Discovery")
  - [ ] 8 stage progress dots — filled dots for completed stages, bright dot for current, gray for upcoming
  - [ ] "Active" status badge (green)
  - [ ] "Decision Maker" role badge (purple)
  - [ ] A "View Project" button (or "Review Definition" / "Review Proposal" if applicable — see Part 3)

### 2D — CTA button links correctly
- [ ] Click "View Project" on the project card
- [ ] Confirm it opens the portal page with `?project_id=X` in the URL
- [ ] Confirm the portal loads that specific project (not "No project found")

### 2E — Invoices section
- [ ] If the invoicing module is active and invoices exist for this organization:
  - [ ] Confirm an "Invoices" section appears below the project cards
  - [ ] Confirm the invoice table shows: Invoice #, Project name, Date, Amount, Status badge, Balance Due, View link
  - [ ] If any invoices have a balance > $0, confirm the orange "Total Outstanding Balance" callout appears
  - [ ] Click a "View" link — confirm it opens the invoice detail page
- [ ] If no invoices exist, confirm the Invoices section does **not** appear (it's hidden when empty)

### 2F — Attention banner
- [ ] Send the project definition for review (see Part 3 steps 3A–3C)
- [ ] Return to the Dashboard page while logged in as the DM
- [ ] Confirm the **amber attention banner** appears at the top: "You have 1 item that needs your attention: Review Definition for [Project Name]"
- [ ] Confirm the "View Project" button has changed to **"Review Definition"** (amber/yellow button)

## PART 3 — Definition Consensus Review System (v1.26 features, now testable)

### 3A — Admin: send for review
- [ ] Log in as admin, go to Expand Site → open the test project → Discovery tab
- [ ] Confirm the definition status badge shows **"Draft"**
- [ ] Confirm the **"Send to Client for Review"** button is visible
- [ ] Click it → set a deadline date (at least a few days out) → click Send
- [ ] Confirm the page reloads and the badge now shows **"Sent for Review"**
- [ ] Confirm the "Send to Client for Review" button is gone (replaced by Lock button)

### 3B — Client portal: DM sees the review UI
- [ ] Log in as the Decision Maker test account
- [ ] Visit the portal page (or click through from the dashboard)
- [ ] Confirm the project loads — **not** "No project found" (this was the bug fixed in v1.27.0)
- [ ] Scroll down to the **Project Definition — Review** section
- [ ] Confirm a **countdown timer** is visible showing time remaining until the deadline
- [ ] Confirm each definition field (Site Description, Primary Goal, etc.) shows as its own block with:
  - [ ] The field value
  - [ ] A "✓ Looks good" button
  - [ ] A "Needs revision" button
  - [ ] A "+ Add comment" toggle

### 3C — Verdict buttons
- [ ] Click "✓ Looks good" on one field — confirm it highlights (turns green/active)
- [ ] Click "Needs revision" on a different field — confirm it highlights (turns amber/active)
- [ ] Click "✓ Looks good" again on the same field — confirm it de-selects "Needs revision" and selects the new one (only one active at a time per field)

### 3D — Comments
- [ ] Click "+ Add comment" on any field
- [ ] Confirm a text area appears
- [ ] Type a comment and submit
- [ ] Confirm the comment appears above the input box with your name and timestamp
- [ ] Click "Reply" on that comment
- [ ] Type a reply and submit
- [ ] Confirm the reply appears indented under the original comment

### 3E — Scroll-depth gate
- [ ] Scroll to the bottom of the definition review section
- [ ] Confirm the **"Submit My Input"** button (for Contributors) or **"Make Final Decision"** section (for DM) is only enabled/visible after you've scrolled past all fields
- [ ] If you haven't scrolled past all fields, confirm you see the message "Scroll through all fields to enable"

### 3F — Admin: verdict summary card appears
- [ ] After submitting verdicts as the DM (step 3C), go back to admin → project → Discovery tab
- [ ] Confirm a **"Stakeholder Verdicts"** card has appeared showing something like "X fields accepted, Y need revision"
- [ ] Confirm it only shows when at least one verdict has been submitted (not before)

### 3G — DM Final Decision
- [ ] Log back in as the DM
- [ ] Scroll to the **"Make Final Decision"** section at the bottom
- [ ] Test **Accept**: click "Accept" → optionally add a note → submit
  - [ ] Confirm the page shows a green **"Definition approved!"** banner
  - [ ] Confirm the verdict buttons are gone (review is closed)
- [ ] *(Reset for next test: admin can re-send for review)*
- [ ] Test **Needs Revision**: click "Needs Revision" → add a note → submit
  - [ ] Confirm a red/amber **"Needs Revision"** banner appears with the DM's note visible

### 3H — Post-decision states (admin)
- [ ] After DM accepts: go to admin → Discovery tab → confirm badge shows **"Client Approved"**
- [ ] After DM rejects: go to admin → Discovery tab → confirm badge shows **"Needs Revision"**
- [ ] Confirm the **"Send to Client for Review"** button reappears after a "Needs Revision" result (so you can revise and re-send)
- [ ] Confirm the **Lock** button is always visible regardless of status
- [ ] Click Lock when status is not "Client Approved" — confirm a confirmation prompt appears before locking

## PART 4 — Regression Check (make sure nothing broke)

- [ ] Expand Site project list loads without errors
- [ ] Open a project detail page — all tabs load (Overview, Stakeholders, Discovery, Branding, Proposal)
- [ ] Client portal loads for a project where definition is **locked** (not pending review) — confirm it shows the read-only definition view, not the review UI
- [ ] Mood board / style direction section still shows template images and vote buttons (if templates exist)
- [ ] Proposal section still shows (if a proposal exists)
- [ ] Invoicing module: client invoices page still loads for a user with an org linked

---

## Quick Reference: What Each Status Shows

| Definition Status | Admin Badge | Portal Shows |
|---|---|---|
| `draft` | Draft | Nothing (definition not sent yet) |
| `pending_review` | Sent for Review | Full consensus UI: countdown, fields, verdicts, comments |
| `approved` | Client Approved | Green approved banner + read-only definition |
| `needs_revision` | Needs Revision | Red banner with DM note + read-only definition |
| `locked` | Locked | Read-only definition (no review UI) |
