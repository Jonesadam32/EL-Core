# v1.18.0 Testing Guide — Organizations & Client Management

> **Time needed:** 15–20 minutes
> **Staging site:** https://qd19d0iehj-staging.wpdns.site/wp-admin
> **What this version changes:** Adds a full client/organization data layer. Projects are now linked to organizations instead of using plain text client names. New Clients admin page with card grid, profile pages, contact management, and organization search autocomplete in project creation.

---

## STEP 1: Upload the Plugin

1. Open your staging site WP Admin
2. Go to **Plugins → Add New → Upload Plugin**
3. Click **Choose File** and select `el-core.zip` from `C:\Github\EL Core\`
4. Click **Install Now**
5. When prompted, click **Replace active with uploaded**

**Check:** You see "Plugin updated successfully." If you see errors, stop and tell me exactly what they say.

6. Go to **Plugins** list and confirm EL Core shows **Version 1.18.0**

---

## STEP 2: Verify Migration Ran

The first time v1.18.0 loads, two things happen automatically:
- Two new database tables are created: `el_organizations` and `el_contacts`
- Existing projects get organizations created from their `client_name` values

1. Go to **EL Core → Clients**
2. If you had existing projects with client names, you should see organization cards already created

**Check:** The Clients page loads. If you had projects, orgs should appear. If this is a fresh install with no projects, you'll see the "No Clients Yet" empty state — that's expected.

## COMMON ISSUES

**Clients page is blank or shows error:**
- The core tables may not have been created. Try deactivating and reactivating the plugin.
- Check the browser console (F12 → Console) for JavaScript errors.

**No organizations after upgrade (expected existing ones):**
- The migration only runs for projects where `client_name` is not empty and `organization_id` is 0.
- If the `organization_id` column hasn't been added yet (schema migration didn't run), try refreshing the admin page.

**Autocomplete doesn't show anything:**
- You need at least 2 characters to trigger the search.
- Check browser console for AJAX errors.
- Make sure `elExpandSiteAdmin` is defined (the script should be localized on Expand Site pages).

**Primary contact not added as stakeholder:**
- The contact must have **Portal Access** checked (a WordPress user account).
- The contact must be marked as **Primary**.
- Only happens on new project creation, not retroactively.

**"Add Contact" modal shows error:**
- Email and First Name are required fields.
- If "Create a WordPress account" is checked, the email must not already belong to another WP user (it will link to the existing one instead).

**Client name shows as plain text, not a link:**
- The project's `organization_id` is 0 (not linked). This happens for projects created before v1.18.0 if the migration didn't find a matching org name.
- Creating a new project with the org search will properly link it.
