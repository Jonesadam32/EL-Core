# EL Core v1.14.1 — Modal-based UX

**Release Date:** February 23, 2026  
**Type:** UX Enhancement  
**Status:** ✅ Ready to Deploy

---

## What Changed

Improved the client portal UX by replacing inline content sections with **compact info cards that open modals**. This eliminates wasted empty space and creates a cleaner, more focused interface.

### Key Improvements

1. **Card-based Interface**
   - Deliverables and Feedback now appear as compact, clickable cards
   - Each card shows an icon, title, and count (e.g., "3 items", "5 comments")
   - Empty states show as "None yet" instead of taking up full sections

2. **Modal Interactions**
   - Click any card to view full content in a modal overlay
   - Modals have smooth slide-in animation
   - Multiple ways to close: close button, overlay click, or Escape key
   - Body scroll locked while modal is open

3. **Removed Redundancy**
   - Removed "Completed" and "Active" badges from stage headers
   - These were redundant with the checkmarks in the stage navigation

4. **Project Definition**
   - Moved to a compact card in the global section
   - Opens in modal instead of inline display
   - Cleaner presentation of project details

### Visual Example

**Before (v1.14.0):**
- Large empty "Deliverables" section with "No deliverables yet" message taking vertical space
- Large empty "Feedback" section with "No feedback yet" message
- Redundant "Completed" badge below stage title

**After (v1.14.1):**
- Compact "Deliverables" card: icon + "None yet" (one line)
- Compact "Feedback" card: icon + "None yet" (one line)
- No redundant badges (checkmark in nav is enough)

## Technical Details

### New Components

**Info Cards** (`.el-es-info-card`)
- Flex layout with icon, content, and arrow
- Hover effects: border color change, shadow, lift animation
- Responsive sizing and touch-friendly spacing

**Modal System** (`.el-es-modal`)
- Full-screen overlay with backdrop blur
- Centered container with max-width: 700px
- Header with title and close button
- Scrollable body for long content
- Smooth slide-in animation

### New CSS

- 150+ lines of modal and card styles
- Responsive breakpoints maintained
- Consistent Modern Tech color palette
- Accessibility: `aria-hidden` attributes, keyboard support

### New JavaScript

- Modal open/close handlers
- Keyboard support (Escape closes modal)
- Body scroll management
- Event delegation for performance

## Notes

This is a **non-breaking change**:
- No database schema changes
- No new AJAX handlers
- Only frontend HTML/CSS/JS updates
- Safe to deploy over v1.14.0 or v1.10.7

**Performance Impact:** Minimal
- Modals load on-demand (HTML rendered but hidden)
- JavaScript uses event delegation
- CSS animations use GPU-accelerated transforms

**Browser Support:**
- Modern browsers (Chrome, Firefox, Safari, Edge)
- Backdrop blur may degrade gracefully on older browsers
- Core functionality works without JavaScript (cards still visible)
