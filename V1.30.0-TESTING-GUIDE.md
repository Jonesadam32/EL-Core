# v1.30.0 Testing Guide

**Plugin:** EL Core v1.30.0  
**Uploaded to staging:** _(upload before testing)_  
**Focus:** Definition Consensus Review — corrected Needs Revision flow, Reset to Draft escape hatch, portal stage names, Qualification stage message.

---

## Before You Start

1. Upload `el-core-v1.30.0.zip` via WordPress Admin → Plugins → Add New → Upload Plugin
2. Verify it says "EL Core 1.30.0" in the plugin list
3. Have at least one test project in stage 2 (Discovery) with a definition sent for review
4. Have two test users: one as DM, one as Contributor (both stakeholders on the project)

---

## Part 1 — Needs Revision Flow (the critical fix)

This is the most important thing to test. The old behavior stranded the client; the new behavior keeps the review open.

### 1A — DM clicks "Needs Revision"

1. Log in as the DM stakeholder on the portal
2. Open the project portal → scroll to the Project Definition review section
3. Confirm the review UI loads (fields, verdict buttons, DM decision section)
4. In the "Make Final Decision" section, type a note in the note box (e.g., "Please clarify the primary goal — it's too vague")
5. Click **"Needs Revision"**

**Expected:**
- [ ] Page does NOT do a full reload
- [ ] The review UI reloads in-place (spinner → re-renders)
- [ ] A yellow/amber **DM revision banner** appears at the top of the review section
- [ ] The banner shows the note you typed: "Decision Maker requested changes: Please clarify the primary goal — it's too vague"
- [ ] The definition fields are **still visible and editable** (Edit buttons present)
- [ ] Verdict buttons are still present (contributors can re-vote)
- [ ] The "Make Final Decision" section is still shown (DM can still click Accept when satisfied)
- [ ] No "Needs Revision" closed state banner — the review is still open

### 1B — Contributor edits a field after DM notes revision

1. Log in as a Contributor
2. Open the same project portal
3. Confirm the DM revision banner is visible
4. Click **"✏ Edit"** next to the Primary Goal field
5. Change the text and save

**Expected:**
- [ ] Field saves without error
- [ ] New value shows immediately
- [ ] DM banner remains visible

### 1C — DM clicks "Accept" after revision

1. Log in back as DM
2. Open the portal — verify the edited field value is shown
3. In the "Make Final Decision" section, optionally add a note, then click **"Accept"**

**Expected:**
- [ ] Full page reload
- [ ] Review section replaced with green "Definition approved!" banner
- [ ] Definition grid (read-only) shown below banner
- [ ] Verdict buttons / DM decision section are gone

### 1D — Admin sees approved state

1. Log in as admin
2. Open the project detail → Phase 2 panel
3. Check the status badge

**Expected:**
- [ ] Status badge shows "Client Approved" (green)
- [ ] Amber lock banner is shown: "The client has approved the Project Definition. Lock it now to proceed."
- [ ] "Lock Definition" button is present
- [ ] "Send to Client for Review" button is NOT shown (definition is approved, not draft)
- [ ] "Reset to Draft" button is NOT shown (definition is approved, not pending_review)

## Part 2 — Admin Reset to Draft Escape Hatch

### 2A — Send definition for review, then reset

1. Log in as admin
2. Navigate to a project in Phase 2 with a definition in `draft` state
3. Click "Send to Client for Review" → set deadline → confirm
4. Page reloads → status badge should show "Sent for Review" (blue)
5. Confirm "Reset to Draft" button is now visible

**Expected:**
- [ ] Status badge: "Sent for Review"
- [ ] "Send to Client for Review" button is GONE (definition is pending_review, not draft)
- [ ] "Reset to Draft" button is visible (ghost/secondary style)

### 2B — Click "Reset to Draft"

1. Click **"Reset to Draft"**
2. A confirmation dialog appears: "Cancel the active client review and return this definition to Draft? The client will no longer see the review UI until you send again."
3. Click OK

**Expected:**
- [ ] Page reloads
- [ ] Status badge: "Draft"
- [ ] "Send to Client for Review" button is back
- [ ] "Reset to Draft" button is gone

### 2C — Verify client portal after reset

1. Log in as a portal user (DM or Contributor)
2. Open the project portal
3. The definition review section should no longer be visible (review is closed/cancelled)

**Expected:**
- [ ] No review UI in the portal — the definition section is gone (no definition cards shown because `review_status = draft`)
- [ ] Or if the definition was previously locked, only the locked view shows

## Part 4 — Qualification Stage Graceful Message

### 4A — Project in stage 1

1. Create a test project or advance a project back to stage 1
2. Log in as a portal user on that project
3. Stage 1 (Qualification) should be the current/selected stage

**Expected:**
- [ ] A blue info banner appears at the top of the stage 1 content area
- [ ] Banner text: "Your project is in the early qualification stage. We're getting to know your goals and confirming this project is a great fit. You'll hear from us shortly to schedule a discovery call."
- [ ] The standard deliverables/feedback cards still appear below the banner
