# EL Core v1.28.0 — Testing Guide

> **Upload first:** WordPress Admin → Plugins → Add New → Upload Plugin
> Use `el-core-v1.28.0.zip` from your Downloads folder (or `releases/`).
> Confirm "Version: 1.28.0" appears in the Plugins list before starting.
>
> **What's in v1.28.0:** 5 bug fixes found during v1.27.x testing (3G/3H):
> - Verdict button now keeps comment toggle working (no refresh needed)
> - Needs-attention project list now catches approved/needs-revision projects
> - Amber lock prompt banner after client approval
> - Admin project detail UX redesign (stage stepper + status card + smart tab activation)
> - Definition version history with field diffs (DB migration v9 — new column added automatically)
>
> **Parts 1–3F were tested and passed in v1.27.x.** This guide starts from 3G
> and covers all the new v1.28.0 features. If you're starting fresh or want
> to verify end-to-end, start from Part 1.

## Quick Reference: Status → What Admin Sees

| Definition Status | Badge | Stage Status Card | Amber Banner? | Needs Attention? |
|---|---|---|---|---|
| `draft` | Draft (gray) | "Draft — not yet sent" | No | No |
| `pending_review` | Sent for Review (blue) | Deadline countdown | No | No |
| `approved` | Client Approved (green) | "Lock required" (amber) | **Yes** | **Yes — Lock Required** |
| `needs_revision` | Needs Revision (amber) | "Revision requested" + DM note (red) | No | **Yes — Needs Revision** |
| `locked` | Locked (green) | "Locked ✓" (green) | No | No |
